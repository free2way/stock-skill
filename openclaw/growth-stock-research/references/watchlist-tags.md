# Watchlist Tags

Use this reference when turning report JSON files into practical watchlist buckets.

## Goal

Make a long watchlist easier to manage by applying transparent, lightweight labels.

## Built-in Tags

The tagger can emit labels such as:

- `high-quality`
- `high-risk`
- `watch-only`
- `needs-review`
- `strong-score`
- `weak-score`
- `missing-data`
- `has-backtest`

## Philosophy

These tags are not forecasts. They are management aids.

- `high-quality` means the current score and data completeness look relatively strong.
- `high-risk` means the report shows weaker scores or obvious risk pressure.
- `watch-only` means the name may be interesting but is not clean enough for action.
- `needs-review` means the report is stale or incomplete enough to deserve manual attention.

## Best Practice

Use tags to narrow attention, then read the actual report before making any decision.
